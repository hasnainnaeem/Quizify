Get Started

Quizify is an application for entrance test preparation. This documentation is intended to help user to understand the source files of Quizify.

Running Source Files

              
               1. Import all the Java class files in any Java Compiler.
	       2. Import provided external libraries in compiler.
	       3. Place folders named "Files" and "Software Files" in source code (src) folder of your project.
	       4. Now in classes named : "ImportMCQ", "Test", "Full Test", "TestProgressChart", "AllTestsProgressChart"
	          replace file paths.DON'T replace full paths, but only until "\\src" folder.
               5. That's it. Code is ready to compile.

Adding MCQs and Configuring Tests

Learn how to add more MCQs in chapters of subjects and how to change number of mcqs in a test.

Add/Remove MCQS In "Files" folder, there are respective folders for each subject. In each folder, there are multiple text files. Each text file name indicates chapter name. To add MCQS, simply write them at the end of the file in a way other MCQS are written. First line is indicates question, next 4 lines are possible options. 5th line is for correct mcq and 6th Line is for explanation.
Change MCQS per Test In MainMenu class, number of mcqs per test can be changed. For HalfBookTest and FullBookTest, MCQS needed from a single chapter can be modified in methods "takeHalfBookTest" and "TakeFullBookTest". For FullTest, mcqs per book can be changed in method "takeFullTest". Number of mcqs in ChapterWiseTest can not be changed. All mcqs are displayed to user from a single chapter in this test. This is due to the reason that mcqs are randomly picked in other tests, if it was to work in same way, user might miss some mcqs during every test.
SoftwareFiles Folder This folder is to store software data. Files in this folder basically store data related to performance of user in tests. "TestDetails.txt" file stores marks in last test attempted by user. This is used to generate a pie chart for user. "TestData" folder includes files to store fraction of correct MCQS attempted by user. For example, if user attempted 8 correct mcqs out of 10. This fractional will be 8/10. These fractions are used to generate Pie chart of user's performance in similiar kind of tests.

Efforts were made to make the code as simple and general as possible. Focus was to write reusable code. Aggregate formulas, options in Main Menu are pretty simple to be changed from respective classes.

External Library and Charts Working

"JFreeChart" external library is used to generate Pie and Line charts. Libarary files are placed with source files. Working of these charts is made pretty simple. These charts import data from text files and generate charts.

For a single test, Pie chart is generated according to data in "SoftwareFiles/TestDetails.txt" file.

For performance tracking of multiple tests, Line chart is used. Which uses data in "SoftwareFiles/TestData" folder.

General Overview of working of Program

There are three main modules in program. Module related to tests works by importing whole data from a specific file(s) depending upon the test type. Then mcqs are one by one presented to user. Before displaying correct MCQ, user's choice input is necessary and correct/wrong mcqs are counted. At the end of the test, number of correct, wrong, total mcqs are is stored in "TestDetails.txt" file. Percentage of correct mcqs is stored in a file in TestData Folder, depending on the test type.

Charts are generated on the basis of the stored data. Charts automatically adjust themselves according to the provided data.

Working of Aggregate Calculator is made pretty simple by filtering out common functionality and making methods with different formulas for each university.
